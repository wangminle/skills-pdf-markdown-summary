## 1. 本轮目标

本轮在此前 DeepSeek V3.2、FunAudio-ASR、GPT-5 System Card、Gemini 2.5 Report 四份 PDF 已调优结果的基础上，新增验证并优化以下三份 PDF：

- `2509.17765v1-Qwen3-Omni_Technical_Report`

## 2. 调试方法

本轮没有只针对单份 PDF 调整固定参数，而是继续完善通用截图流程：

1. 使用 `--preset robust --debug-visual` 跑新增三份 PDF，查看截图、画线阶段输出和总览图。
2. 将问题按“caption 锚点、方向判断、文本边界、表格连续行带、正文污染”分类。
3. 为每类真实失败形态补充小型回归测试，再修改通用算法。
4. 每轮修改后先跑定向测试，再重跑有问题的 PDF。
5. 最终同时重跑新增三份 PDF和既有四份 PDF，并对七份 PDF 的总览图进行目视检查。
6. 对既有四份 PDF 与 `tests/results/20260606-022/` 做文件数量和 SHA-256 差异对照，用于识别潜在回退；对于哈希变化的截图，再以实际图像内容判断是否为更完整或更干净的合理变化。

## 3. 基线问题与根因

### 3.1 Qwen3-Omni Technical Report

基线输出暴露了本轮最多的边界问题：

- Table 9 未导出。
- Tables 10 至 13 被截断，表格底部行缺失。
- Table 3 混入表格之后的正文。
- 部分短表、数字密集表格行未被连续行带识别。

根因包括：

- 显式 `Table N:` caption 位于较长文本块中时，被正文引用过滤逻辑误拒绝。
- 连续表格行带只接受强结构行，未充分覆盖短数字单元格和分组标题行。
- 表格结束边界对正文与弱表格行的区分不够稳定。

### 3.2 新增三份 PDF 的通用图示边界

Attention 和 HFT 文档中存在短标题、图内标签、紧凑数字块等结构。此前通用正文边界阻断容易把这些真实图示内容当作正文，从而过早截断。

根因是短文本块只按宽度、位置和距离判断，没有结合截图方向、相邻文本块形态和标签聚集程度。

### 3.3 四份既有 PDF 的回归风险

在放宽短标签和数字块保护的中间迭代中，DeepSeek Figure 1 和 Table 1 曾出现边界退化。说明“允许短文本继续扩展”不能无条件生效，必须限制到真正像图内标签或紧凑表格行的结构。

## 4. 算法调整

### 4.1 改进表格连续行带裁剪

调整 `refine_clip_to_table_band()`：

- 按 X 坐标顺序汇总行内文本块，稳定识别多单元格结构。
- 分组标题或桥接行只有在后方仍存在表格证据时才允许进入行带。
- 允许紧凑的数字型弱表格行继续扩展表格边界。
- 数字型弱表格行宽度限制为候选截图宽度的 70% 以内，避免将横跨整行的数字正文误当表格。
- 保留对编号式章节标题的排除，防止章节标题被拼入表格。

该调整解决了 Qwen Tables 10 至 13 的底部截断，并清除了 Table 3 后方正文。

### 4.2 修正显式 caption 的正文引用判定

调整 `is_caption_reference()`：

- 显式 `Table N:` / `Figure N:` 冒号 caption 不再仅因所在文本块较长就被判定为正文引用。
- 普通正文中的 `Table N, ...`、`Figure N shows ...` 等引用仍继续过滤。

该调整恢复了 Qwen Table 9，同时没有重新引入 Gemini Table 4 的正文引用误识别。

### 4.3 改进正文边界阻断与图内标签保护

调整 `limit_clip_by_text_blocks()`：

- 短标题保护改为与截图方向相关，避免远离内容的孤立标题被错误保留。
- 短标题后紧接正文块时，仍在标题处阻断截图。
- 对邻近区域内三个及以上的短图示标签进行聚类保护，避免流程图或示意图内标签被当成正文。
- 对宽度不超过候选区域 75% 的紧凑数字块提供保护。
- 横跨整行的数字正文仍作为边界阻断证据。

该调整在保护 Attention、HFT 图示内容的同时，修复了中间迭代对 DeepSeek 的回归。

### 4.4 调用链同步

Figure 和 Table 提取路径均传入截图方向与必要上下文，使正文边界逻辑能区分“caption 上方截图”和“caption 下方截图”，避免同一规则在相反方向中产生不一致结果。

## 5. 回归测试

在 `tests/scripts/test_caption_anchor_quality.py` 中补充了本轮真实失败形态的测试，覆盖：

- 显式冒号 caption 位于长文本块时仍应保留。
- 表格分组标题后存在真实表格行时允许进入连续行带。
- 紧凑数字表格行可继续扩展表格边界。
- 全宽数字正文不得被当作表格行。
- 编号式章节标题不得进入表格截图。
- 孤立远端短标题应阻断截图。
- 短标题后紧接正文时应在标题处停止。
- 多个邻近短标签应作为图示内容保留。
- 紧凑数字图示块与全宽数字正文的区分。

## 8. 最终结论

本轮效果提升主要来自通用流程和判定规则的完善，而不是针对三份新 PDF 调整专用参数。最终算法能够更稳定地区分：

- 显式 caption 与正文引用；
- 表格弱结构行与普通正文；
- 图内短标签与章节标题；
- 紧凑数字内容与全宽数字段落；
- 合理边界变化与真实回归。
