# PDF 双栏与强结构表格裁剪调优记录（2026-06-13）

## 1. 问题与结论

本轮复核并修复 BUG-018：

- 双栏论文中，`limit_clip_by_text_blocks()` 只按 Y 坐标筛选 blocker，另一栏标题可能提前终止当前栏图表裁剪。
- 强结构表格中，`refine_clip_to_table_band()` 对所有已开始的行带无条件应用 `max_row_gap`，分组留白可能导致后续真实表格行被截断。

两条 review comment 均属实。仅取消强结构表格的行距终止会引入新问题，因此最终实现同时约束了栏位相关性、行结构和句子型正文特征。

## 2. 最终修复

### 2.1 双栏 blocker 候选过滤

对明显窄栏的 caption：

- 优先读取 `TextBlock.column` 或其文本单元的唯一栏位。
- 栏位信息不可用时，使用 caption 水平扩展带的重叠关系兜底。
- 排除另一栏标题和正文块，避免它们成为当前栏的 blocker。

对宽 caption 不启用栏位过滤，保持跨栏图表和表格原有候选集合。

### 2.2 强结构表格分组间距桥接

较大行间距不再无条件结束强结构表格行带。只有同时满足以下条件时才允许桥接：

- 前后行均为强结构行。
- 间距仍在 `max_bridge_gap` 内。
- 后一行不是句子型正文。

句子型正文结合字符数、单词数和数字证据判断。该限制可以保留真实分组末行，同时阻止被拆成多个水平块的正文误并入表格。

- 强结构表格应跨越分组留白并保留后续真实行。
- 大间距后的句子型强行不得桥接进表格。

## 5. 调试过程中的关键发现

初始修法将强结构表格完全排除在 `max_row_gap` 终止之外，真实 Attention 测试立即暴露两个回归：

- Table 2 被截为上半部分。
- Table 3 混入下方正文。

原因是 PDF 正文可能被拆成两个水平文本块，从而被误分类为强结构行。真实画线验证表明，修复不能只依据 `strong` 标签，必须结合句子型特征和数字结构证据。

## 6. 最终结论

BUG-018 已完成代码修复、回归测试和真实双栏 PDF 画线验证。最终规则能够保留强结构表格的分组行带，同时排除另一栏 blocker 和表格后的句子型正文，未发现 Attention 图表截图退化。
